# 9. Qwen3-family sampling — greedy default, opt-in env-var overrides only

Date: 2026-05-24

## Status

Accepted.

## Context

Phase 13 of the ROADMAP focused on Qwen3-family-tuned prompt-library optimisation. Alibaba's published recommendation for Qwen3 instruct / non-thinking inference is `temperature=0.7`, `top_p=0.8`, `top_k=20`, `presence_penalty=1.3` — a profile designed to introduce lexical variety while keeping the model on-task. The original Phase 13 framing assumed adopting the recommended profile by default would improve output quality across the recipe library, with `scripts/delegate.sh` and `experiments/runner.sh` auto-applying the profile whenever the resolved model name matched a Qwen3-family substring (`qwen3.6`, `qwen3-coder`, `qwen3-next`, `qwen3.5`). The implementation landed in an earlier commit on the Phase 13 branch (`31f244e`) with the four sampling knobs surfaced as env vars `DELEGATE_TEMPERATURE`, `DELEGATE_TOP_P`, `DELEGATE_TOP_K`, `DELEGATE_PRESENCE_PENALTY` so callers could opt out per-call if needed.

An A/B against the T4 commit-message fixture on `qwen3.6:35b-a3b-q8_0` was run before the PR opened, with five reps per arm against the 2026-05-21 fixture (the calibration anchor the recipe was tuned against). The full results live in `experiments/results/2026-05-22-track-a-qwen-sampling-ab.md`. The baseline arm (`DELEGATE_TEMPERATURE=0`, greedy decoding) scored `T4_SUMMARY: reps=5 total_passed=30 total_checks=30 mean=1.0000 stdev=0.0000 min=1.0000 max=1.0000` — every guard binding on every rep, deterministic. The treatment arm (Alibaba's recommended Qwen profile auto-applied) scored `T4_SUMMARY: reps=5 total_passed=26 total_checks=30 mean=0.8666 stdev=0.0667 min=0.8333 max=1.0000` — a regression of -0.13 on the mean, with three of the five reps emitting participial-padding tails (`, allowing direct comparison…`-style) that the commit-message recipe's anti-padding directive explicitly forbids, and one rep exceeding the 72-character subject ceiling.

The mechanism is straightforward to read off the numbers. Greedy decoding at `temperature=0` produces verbatim deterministic output — the same commit message lands across all five reps, and the recipe's calibration is anchored against exactly that shape. Alibaba's `temperature=0.7 + presence_penalty=1.3` profile reintroduces lexical variety, and in this fixture's prompt shape the variety tends to land on the exact participial-form restatement tails the anti-padding directive was designed to block. The calibration binds at `temperature=0` but breaks at `temperature=0.7`, and the failure shape is precisely the one the recipe spent three rounds of MISS evidence (2026-05-10, 2026-05-11, 2026-05-12) tightening guards against. The user's auto-memory at `project_qwen3_greedy_default.md` records the headline finding as "Qwen3 greedy decoding is the right default" — empirically beats Alibaba's recommended profile on T4 (1.00 → 0.87 regression).

## Decision

Greedy decoding (`temperature=0`) is the default sampler for all models in `scripts/delegate.sh`, including Qwen3-family models, regardless of Alibaba's published recommendation. The four sampling knobs surface as opt-in env vars only: `DELEGATE_TEMPERATURE`, `DELEGATE_TOP_P`, `DELEGATE_TOP_K`, `DELEGATE_PRESENCE_PENALTY`. When all four are unset, the wrapper sends `temperature:0` to the backend (Ollama `/api/generate` or MLX `/v1/chat/completions`) and the model produces deterministic output. When any of the four is set, the wrapper forwards each set value into the backend payload and emits the resolved values into the metrics JSONL line as `sampling_temperature`, `sampling_top_p`, `sampling_top_k`, `sampling_presence_penalty` so post-hoc analysis can correlate sampling regime to per-call outcome.

The treatment-2 measurement after the default flip (`T4_SUMMARY: reps=5 total_passed=30 total_checks=30 mean=1.0000 stdev=0.0000 min=1.0000 max=1.0000`) confirms the regression was sampler-induced rather than caused by an unrelated recipe or fixture change — the same recipe template, same fixture, same model, scored back to 1.00 mean once greedy was restored as the default. The env-var surface remains useful for callers who want to opt into the Qwen profile for a specific call where lexical variety helps (free-form prose drafting, brainstorming, paraphrase generation) — the default just no longer applies it implicitly to every Qwen3-family delegation.

The mirror change in `experiments/runner.sh` keeps the production code path and the measurement path in lock-step. Both consume the same env-var-driven sampler-resolution function, so a future A/B can be re-run by setting `DELEGATE_TEMPERATURE=0.7 DELEGATE_TOP_P=0.8 DELEGATE_TOP_K=20 DELEGATE_PRESENCE_PENALTY=1.3 bash experiments/runner.sh --reps 5 qwen3.6:35b-a3b-q8_0` against the current commit-message fixture without needing to patch any code.

The rejected alternative was keeping the Qwen profile as default and tightening the commit-message recipe guards further to block the new participial-padding shapes the treatment introduced. Rejected because the regression was not bounded to one shape — three of five reps emitted padding tails and one exceeded the subject ceiling, suggesting the sampler-induced variety would keep surfacing new failure shapes faster than the directive enumeration could absorb them. The directive-binding ceiling (later documented as Phase 15 / `prompts/README.md` Convention 3) was already a known phenomenon at this point; extending the enumeration to chase sampler-induced variety would have stacked an open-ended calibration cost on top of a default that delivered no measured quality benefit on the T4 axis.

Phase 13 shipped via PR [#196](https://github.com/IsmaelMartinez/delegate-to-ollama/pull/196) (squash `1f0a86d`, merged 2026-05-23), with the default-flip-back commit (`1e8909e`) landing in the same PR.

## Consequences

The recipe library's calibration history stays binding because the default sampler stays at `temperature=0`. Every guard line in `prompts/<task>.md` is tied to a dated observation made under greedy decoding; flipping the default to a stochastic profile would have silently invalidated that calibration without flagging which guards stopped binding. Recipes that calibrated to 1.00 mean on the T4 / T5 / T6 fixtures stay at 1.00 mean for new callers who do not override the sampler.

The env-var surface is the right shape for callers who genuinely want sampling variance. A caller drafting brainstorm prose where determinism actively hurts (every rerun produces the same output) can set `DELEGATE_TEMPERATURE=0.7` for that call and get the Qwen profile's variety; the metrics row records the override so the calibration history can still attribute outcomes to sampler regime. Recipes whose `## Calibration notes` are explicit about being measured under greedy are protected from a caller silently breaking the binding by setting the env var — the convention is that recipe authors who want a different sampler call it out in the recipe's `## Invocation` example.

What would justify revisiting the decision: an A/B against the current T4/T5/T6/T7/T8 scorer set showing the Qwen profile (or any other published profile) materially improves mean score on the recipe shapes the skill actually serves, with the improvement holding under deterministic reps. The current A/B measured the opposite on T4; if a future round measures the opposite of the opposite (Qwen profile improves T4 by ≥0.05 mean with non-overlapping stdev intervals against greedy), the default flip should be reconsidered. The measurement itself is cheap — five reps per arm runs in under two minutes on the reference host — so the trigger for re-measuring is low.

The cost of this ADR is one more file in `docs/adr/`. The benefit is that the empirical evidence justifying greedy-as-default — the A/B numbers, the failure mechanism, and the explicit rejection of the auto-apply-the-published-profile path — now lives in a durable single artifact alongside `experiments/results/2026-05-22-track-a-qwen-sampling-ab.md` rather than only in the ROADMAP Phase 13 narrative. A future contributor who reads Alibaba's recommendation and wonders why this skill ignores it finds the measurement here rather than having to re-derive the regression.
