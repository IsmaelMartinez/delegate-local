# Local-model accuracy baseline — 2026-04-28

First measured comparison of four installed Ollama models against three fixture tasks. Inputs and prompts under `experiments/fixtures/`, raw outputs under `experiments/results/raw/`. All runs initiated from a single workstation (Apple M5 Max, 128 GB unified memory).

## Models tested

| Model                                  | Tier (per pick-model.sh)         | On-disk size |
|----------------------------------------|----------------------------------|--------------|
| `phi4-reasoning:plus`                  | reasoning                        | 11 GB        |
| `gemma4:31b-it-q8_0`                   | prose / long-context fallback    | 33 GB        |
| `qwen3.6:35b-a3b-q8_0`                 | prose / long-context fallback    | 38 GB        |
| `qwen3-next:80b-a3b-instruct-q8_0`     | prose (current default)          | 84 GB        |

Combined active weight (166 GB) exceeded available unified memory, so the first attempted parallel run produced memory swapping; the data below comes from a sequential re-run with one model resident at a time.

## Tasks

| ID  | Task                          | Style                   | Expected output       |
|-----|-------------------------------|-------------------------|-----------------------|
| T1  | Doc-drift verification        | Closed structured       | Four `CLAIM_X: <verdict>` lines; ground truth A=still-open, B/C/D=already-done |
| T2  | Party-config structural check | Closed structured       | `CLEAN` (the 6 YAMLs share the expected shape) |
| T3  | Loose-end pattern review      | Open-ended inference    | `NONE` is the safe answer; any concern raised must name a real, checkable target |

T1 and T2 have a single ground-truth answer. T3 is the deliberate hallucination-test: prior session work showed that an open-ended "find issues" prompt over commit subjects + diffstats invites invented findings.

## Scoring rubric

- **T1**: 1 point per claim correctly classified, max 4. The roadmap entry for D ("deep-freeze … or don't") accepts either verdict; both `still-open` and `already-done` count as correct.
- **T2**: 1 if exactly `CLEAN`, 0 if anything else. The fixture has no structural deviations, so any reported deviation is invented.
- **T3**: For each concern raised, scored independently as **real** (1.0), **plausible-but-resolved** (0.25), or **hallucinated/format-broken** (0.0). Score = sum / count. `NONE` would have scored 1.0.

## Results

| Model                                  | T1 score | T2 score | T3 score | Wall time (s) |
|----------------------------------------|----------|----------|----------|---------------|
| `gemma4:31b-it-q8_0`                   | 4/4      | CLEAN ✓  | 0.56     | 741           |
| `phi4-reasoning:plus`                  | 4/4      | CLEAN ✓  | 0.06     | 748           |
| `qwen3.6:35b-a3b-q8_0`                 | 4/4      | CLEAN ✓  | 0.06     | 331           |
| `qwen3-next:80b-a3b-instruct-q8_0`     | 4/4      | INCONSISTENT ✗ | 0.00 | 256       |

All four models matched on T1 (closed structured task with explicit format spec) — including verdict on the deliberately-ambiguous claim D, which every model classified as `already-done`.

T2 separated the field. Three models returned the requested single-token `CLEAN`. `qwen3-next:80b` interpreted the prompt loosely and produced a long content review claiming the SNP and Labour climate position values were inconsistent with their stance text — the prompt asked for *structural* deviations only and explicitly said "do not invent issues". This is the same failure mode that broke its earlier merge-pattern review in the parent session: the largest model is the most prone to going off-prompt.

T3 is where everything fell apart:
- `qwen3-next:80b` finished in 5 seconds and emitted four lines reading literally `CONCERN | <path>` with no concern text — a format violation that a calibrated reader would reject outright.
- `phi4-reasoning:plus` and `qwen3.6:35b` each raised four concerns, all of which named *already-fixed* targets: PR-shipped IGV branding fixes, the merged region-card count fix, news.json automation that exists, etc. The model treated each commit subject as describing an *open* problem rather than a *closed fix*. Both produced extensive chain-of-thought confidence around findings that fall apart on a single `git log` check.
- `gemma4:31b` was the only model whose concerns held up on verification. Two of its four pointed at real loose ends (more `simplify-` phases planned in `docs/superpowers/plans/2026-04-28-simplify.md`; the post-election items still open in `ROADMAP.md`), one was speculative-but-checkable (the dev preview page exists; whether to remove it is a judgement call), and one already-fixed.

The headline finding from T3 is that no model approached `NONE` as a valid answer despite the prompt offering it. All four felt obliged to fill the slot. This matches the observed hallucination pattern from the parent session and is the strongest argument in the skill's red-flag list.

## Speed observations

`qwen3-next:80b` and `qwen3.6:35b` were materially faster than `phi4-reasoning:plus` and `gemma4:31b`. The reasoning-tier models pay for accuracy with wall-clock, especially on T2 where `phi4-reasoning` spent 7+ minutes producing chain-of-thought before emitting `CLEAN` — a single token answer. For trivial structural checks, reasoning-tier is overkill.

`qwen3.6:35b` is the speed/accuracy sweet spot for the closed tasks: matched `phi4-reasoning` on T1 and T2 in roughly half the wall time. Its T3 output was no worse than `phi4-reasoning`'s.

## Implications for tier routing

| Tier         | Recommended after this run                                                   |
|--------------|------------------------------------------------------------------------------|
| `code`       | Unchanged (not tested here).                                                 |
| `prose`      | Promote `qwen3.6:35b-a3b-q8_0` ahead of `qwen3-next:80b`. The 80B model's tendency to interpret prompts loosely (T2) and emit format-broken output (T3) outweighs its size. |
| `reasoning`  | Keep `phi4-reasoning:plus` for closed structured tasks where chain-of-thought is acceptable. Accuracy was perfect on T1 and T2. |
| `long-context` | `qwen3.6:35b` confirmed as a reasonable long-context choice; ahead of `qwen3-next:80b` on closed tasks at half the wall time. |

The bigger conclusion is that *prompt scope* dominates *model size*: all four models scored identically on T1, three of four on T2, and the open-ended T3 punished every model regardless of size or tier. The skill body now reflects this in the new "Prompt scope — closed vs open" section.
