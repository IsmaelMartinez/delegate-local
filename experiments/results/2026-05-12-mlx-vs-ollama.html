Read this before drawing conclusions

The 15× headline speedup is mostly a methodology artifact. Ollama runs T1–T3 through the ollama run CLI with reasoning enabled, which streams the whole think-aloud trace. MLX runs every task through /v1/chat/completions with enable_thinking:false. The fair apples-to-apples comparison is the T4–T6 row: same API, same think-off — MLX wins by ~1.8×, not 15×.

The T3 0.00 score on MLX is a scorer-format mismatch, not a hallucination gap. MLX produced exactly the 4 pattern lines the prompt requested, but wrapped each path in backticks (`src/pages/index.astro`). The deterministic scorer's literal-substring check fails on the wrapping characters even though the underlying paths are real. Ollama's 0.47 is partly real, partly artifact of reasoning leak inflating the claim count to 101 across 3 reps.

T4's 5/6 on MLX is a real recipe miss. All three reps ended their commit body with the trailing cliché "This closes the gap between asserted hardening and measured accuracy", which the recipe's BODY_NO_PADDING regex catches. The fix belongs in prompts/commit-message.md: promote the "no closes-the-gap / closes-the-loop tail" guard from a scorer-only check into a directive in the template body with a contrastive Wrong/Correct one-shot.

What this run actually validates. PR #112's /v1/chat/completions switch produces real, schema-correct, instruction-following output on the production prose-tier 35B model. T5 (JSON schema) and T6 (regex acceptance tests) — the two most rigid rubrics — score perfectly on both backends. MLX is the correct backend choice for memory-constrained Apple Silicon setups; the speed advantage is real but modest, not order-of-magnitude.
